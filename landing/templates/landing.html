How this site is structured and how Django works

Structure

The urls.py file forwards the appropriate view to the views.py file based on a HTTP request. The models.py file reads and writes data with the views.py file. The templates provide the HTML to the views.py file. The views.py file then gives a HTTP response, which is the desired website with the desired format/view.

Django

Django does all of the tedious tasks when setting up a web server and website, and uses a friendly programming language: Python. It does this by having a series of components that work together. One of these is the request/response system, which allows Django to accepts requests for URLs and return the HTML needed to display the page. Django passes on each URL to a view so that it can be processed and returned. A view is some Python code that is executed when a URL is accessed, such as returning a line of text, analysing an input etc. Then a response is sent back to the user. This response is usually a HTML web page that are created using templates. With all of the components, creating websites is made easy and structured, which is why famous sites like Instagram use the Django framework.
